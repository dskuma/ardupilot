561/*
562 we use a fixed allocation of BDMA streams. We previously dynamically
563 allocated these, but bugs in the chip make that unreliable. This is
564 a tested set of allocations that is known to work on boards that are
565 using all 3 of ADC3, I2C4 and SPI6. They are the only peripherals
566 that can use BDMA, so fixed allocation is possible as we have 8
567 streams and a maximum of 6 needed.
568
569 The layout is chosen to:
570
571 - avoid stream 0, as this doesn't work on ADC3 or SPI6_RX for no known reason
572 - leave a gap between the peripheral types, as we have previously found that we sometimes
573 lost SPI6 BDMA completion interrupts if SPI6 and I2c4 are neighbours
574 */
575#define STM32_I2C_I2C4_RX_BDMA_STREAM 1
