16/*
17 This driver supports communicating with Torqeedo motors that implement the "TQ Bus" protocol
18 which includes the Ultralight, Cruise 2.0 R, Cruise 4.0 R, Travel 503, Travel 1003 and Cruise 10kW
19
20 The autopilot should be connected either to the battery's tiller connector or directly to the motor
21 as described on the ArduPilot wiki. https://ardupilot.org/rover/docs/common-torqeedo.html
22 TQ Bus is a serial protocol over RS-485 meaning that a serial to RS-485 converter is required.
23
24 Tiller connection: Autopilot <-> Battery (master) <-> Motor
25 Motor connection: Autopilot (master) <-> Motor
26
27 Communication between the components is always initiated by the master with replies sent within 25ms
28
29 Example "Remote (0x01)" reply message to allow tiller to control motor speed
30 Byte Field Definition Example Value Comments
31 ---------------------------------------------------------------------------------
32 byte 0 Header 0xAC
33 byte 1 TargetAddress 0x00 see MsgAddress enum
34 byte 2 Message ID 0x00 only master populates this. replies have this set to zero
35 byte 3 Flags 0x05 bit0=pin present, bit2=motor speed valid
36 byte 4 Status 0x00 0x20 if byte3=4, 0x0 is byte3=5
37 byte 5 Motor Speed MSB ---- Motor Speed MSB (-1000 to +1000)
38 byte 6 Motor Speed LSB ---- Motor Speed LSB (-1000 to +1000)
39 byte 7 CRC-Maxim ---- CRC-Maxim value
40 byte 8 Footer 0xAD
41
42 More details of the TQ Bus protocol are available from Torqeedo after signing an NDA.
43 */
