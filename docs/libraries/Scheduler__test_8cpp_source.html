50/*
51 scheduler table - all regular tasks should be listed here.
52
53 All entries in this table must be ordered by priority.
54
55 This table is interleaved with the table in AP_Vehicle to determine
56 the order in which tasks are run. Convenience methods SCHED_TASK
57 and SCHED_TASK_CLASS are provided to build entries in this structure:
58
59SCHED_TASK arguments:
60 - name of static function to call
61 - rate (in Hertz) at which the function should be called
62 - expected time (in MicroSeconds) that the function should take to run
63 - priority (0 through 255, lower number meaning higher priority)
64
65SCHED_TASK_CLASS arguments:
66 - class name of method to be called
67 - instance on which to call the method
68 - method to call on that instance
69 - rate (in Hertz) at which the method should be called
70 - expected time (in MicroSeconds) that the method should take to run
71 - priority (0 through 255, lower number meaning higher priority)
72
73 */
74const AP_Scheduler::Task SchedTest::scheduler_tasks[] = {
75 SCHED_TASK(ins_update, 50, 1000, 3),
76 SCHED_TASK(one_hz_print, 1, 1000, 6),
77 SCHED_TASK(five_second_call, 0.2, 1800, 9),
78};
