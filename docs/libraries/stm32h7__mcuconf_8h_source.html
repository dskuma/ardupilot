661/*
662 we use a fixed allocation of BDMA streams. We previously dynamically
663 allocated these, but bugs in the chip make that unreliable. This is
664 a tested set of allocations that is known to work on boards that are
665 using all 3 of ADC3, I2C4 and SPI6. They are the only peripherals
666 that can use BDMA, so fixed allocation is possible as we have 8
667 streams and a maximum of 6 needed.
668
669 The layout is chosen to:
670
671 - avoid stream 0, as this doesn't work on ADC3 or SPI6_RX for no known reason
672 - leave a gap between the peripheral types, as we have previously found that we sometimes
673 lost SPI6 BDMA completion interrupts if SPI6 and I2c4 are neighbours
674 */
