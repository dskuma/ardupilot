9// MissionItemProtocol objects are used for transfering missions from
10// a GCS to ArduPilot and vice-versa.
11//
12// There exists one MissionItemProtocol instance for each of the types
13// of item that might be transfered - e.g. MissionItemProtocol_Rally
14// for rally point uploads. These objects are static in GCS_MAVLINK
15// and used by all of the backends.
16//
17// While prompting the GCS for items required to complete the mission,
18// a link is stored to the link the MissionItemProtocol should send
19// requests out on, and the "receiving" boolean is true. In this
20// state downloading of items (and the item count!) is blocked.
21// Starting of uploads (for the same protocol) is also blocked -
22// essentially the GCS uploading a set of items (e.g. a mission) has a
23// mutex over the mission.
