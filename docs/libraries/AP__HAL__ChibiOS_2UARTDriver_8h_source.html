100 /*
101 return timestamp estimate in microseconds for when the start of
102 a nbytes packet arrived on the uart. This should be treated as a
103 time constraint, not an exact time. It is guaranteed that the
104 packet did not start being received after this time, but it
105 could have been in a system buffer before the returned time.
106
107 This takes account of the baudrate of the link. For transports
108 that have no baudrate (such as USB) the time estimate may be
109 less accurate.
110
111 A return value of zero means the HAL does not support this API
112 */
113 uint64_t receive_time_constraint_us(uint16_t nbytes) override;
